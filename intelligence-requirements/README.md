# Intelligence Requirements

This folder contains Priority Intelligence Requirements (PIRs) and the analytical framework used to develop them. PIRs are the foundation of a CTI program — formally stated intelligence questions that drive collection, analysis, and production activity toward decisions that matter to the organization.

---

## Contents

| Document | Description |
|---|---|
| [PIR Development Framework](./pir-development-framework.md) | Methodology for developing, prioritizing, and maintaining PIRs across organizational contexts |
| [Blue Stripe Tech — Active PIR Set](./blue-stripe-tech-pirs.md) | Worked PIR example for a fictional DoD contractor — five active requirements covering nation-state, ransomware, exfiltration, supply chain, and insider threat |

---

## How These Documents Connect to the Portfolio

The Blue Stripe Tech PIR set is the intelligence tasking document that drives every other product in this portfolio. The Volt Typhoon threat actor profile answers PIR 1. The Play ransomware TIC brief answers PIR 2. The phishing investigation addresses PIR 1 and PIR 3. The defensive architecture analysis operationalizes PIRs 1, 3, and 4 into control recommendations.

A PIR set without finished intelligence products is just a list of questions. Finished intelligence products without PIRs are answers to questions no one asked. This folder establishes the analytical thread that connects both.
